# L4 reference answer key — the queue worker lifecycle subsystem

Pinned source: `experiment/repos/laravel` (SHA 2107d3d7…, PHP). All cites re-verified.

A bounded cooperating cluster in `src/Illuminate/Queue/Worker.php` and
`src/Illuminate/Queue/Jobs/Job.php`, with several interrelating paths (the run loop, the
per-job fire path, the exception/retry/fail path) coordinated by the events it raises.

## Required spine (gradeable core — must hit all)

1. **The long-lived loop — `Worker::daemon($connectionName, $queue, WorkerOptions $options)`**
   — `src/Illuminate/Queue/Worker.php:194`. It starts signal handling, records the
   last-restart timestamp, raises `WorkerStarting` (`:212`), and loops (`while (true)`,
   `:214`): checks `daemonShouldRun` (`:222`), gets the next job, fires it, then checks
   `stopIfNecessary` (`:266`).
2. **"Should the loop keep running" gate — `daemonShouldRun`** — `Worker.php:344` (maintenance
   mode / paused); when false the worker `pauseWorker`s and may `stop` (`:224-231`).
3. **Obtaining the next job — `getNextJob($connection, $queue)`** — `Worker.php:425`, called
   from the loop via `$this->manager->connection($connectionName)` (`:243`). A null job →
   `WorkerIdle` event + `sleep`; a real job → `runJob` (`:251`).
4. **Per-job entry — `runJob` → `process`** — `runJob($job, $connectionName, $options)`
   `Worker.php:494` calls `process($connectionName, $job, $options)` (`:498`) inside a
   try/catch that reports and handles lost-connection (`:500-507`).
5. **Firing the job and the events around it — `process`** — `Worker.php:534`. It raises
   `JobProcessing` via `raiseBeforeJobEvent` (`:543`/def `:805`), checks max-attempts, and if
   not deleted calls `$job->fire()` (`:555`); on success it raises `JobProcessed` via
   `raiseAfterJobEvent` (`:557`/def `:819`); in a `finally` it dispatches `JobAttempted`
   (`:566-568`).
6. **The job actually runs — `Job::fire()`** — `src/Illuminate/Queue/Jobs/Job.php:96`. It reads
   the payload, parses the handler name via `JobName::parse($payload['job'])`
   (`src/Illuminate/Queue/Jobs/JobName.php:15`), resolves the handler from the container
   (`Job::resolve`, `Job.php:264` → `$this->container->make($class)`, `:277`) and calls
   `$this->instance->{$method}($this, $payload['data'])` (`:101`).
7. **Exception → retry-or-fail — `handleJobException`** — `Worker.php:578`. If the job has not
   already failed it runs the max-attempts / max-exceptions / no-retry checks
   (`markJobAsFailedIfWillExceedMaxAttempts` `:659`, `…MaxExceptions` `:680`,
   `markJobAsFailedIfItShouldntBeRetried` `:706`), raises `JobExceptionOccurred`
   (`raiseExceptionOccurredJobEvent`, `:609`/def `:834`); in the `finally`, if the job is not
   deleted/released/failed, it computes `$this->calculateBackoff($job, $options)` (`:620`,
   def `:747`), `$job->release($backoff)` (`:623`), and dispatches `JobReleasedAfterException`
   (`:625-627`); then re-throws (`:630`).
8. **Permanent failure path — `Job::failed` / `JobFailed` event** — `Job::failed($e)`
   `Job.php:247` resolves the handler and calls its `failed(...)` if present (`:253-255`);
   the worker's failure marking (`markJobAsFailed*` at `:631`/`:659`/`:680`/`:706`/`:721`)
   leads to the `JobFailed` event (`src/Illuminate/Queue/Events/JobFailed.php`) and
   `$job->fail(...)`, which records the failure for the queue connection.
9. **Stop conditions — `stopIfNecessary`** — `Worker.php:379`: memory limit, queue-restart
   signal (`queueShouldRestart`), max-jobs/max-time/idle-timeout, and lost-connection
   (`stopWorkerIfLostConnection` `:517`) all funnel into `stop($status, $options, $reason)`
   (`:961`).

## Common misses / wrong turns

- Describing only `process`/`fire` and ignoring the enclosing `daemon` loop and its
  keep-running / stop decisions — the prompt asks about the long-lived worker, not just one
  job.
- Stopping the failure story at "the job throws" and never reaching the retry/backoff
  (`calculateBackoff` + `release`, `:620-623`) or the permanent-failure marking + `JobFailed`
  event.
- Conflating `release` (re-queue for retry, `Job::release` `Job.php:131`, sets `$released`) with
  `delete` (`Job::delete` `Job.php:110`, sets `$deleted`) and `fail` — these are the three
  distinct terminal states the worker distinguishes.
- Missing the events the worker raises (`JobProcessing`/`JobProcessed`/`JobExceptionOccurred`/
  `JobReleasedAfterException`/`JobFailed`/`WorkerStarting`/`WorkerIdle`) — they are the
  coordination channel the prompt names ("what events are raised around it").
- Pointing at `Queue::push`/the dispatch side (Bus) instead of the worker consumption side —
  that is the *producer*, not this subsystem.

## Completeness scale

- **Full:** describes the `daemon` loop, `daemonShouldRun`/`stopIfNecessary` keep-running
  decisions, `getNextJob`→`runJob`→`process`→`$job->fire()` run path with the
  before/after events, and the `handleJobException`→retry-with-backoff-or-fail path with
  `JobFailed`, with cites.
- **Partial:** covers `process`/`fire` and the exception path but omits the loop / stop
  conditions, OR covers the loop but leaves the retry-vs-fail decision vague, OR misses the
  events.

  omits the failure/retry machinery.

## Model answer

`Worker::daemon($connectionName, $queue, $options)` (`Worker.php:194`) is the long-lived loop.
On each iteration it checks `daemonShouldRun` (`:344`, maintenance/pause) and pauses or stops
if not (`:222-231`); otherwise it fetches the next job via `getNextJob` (`:425`) from the queue
connection. A null job dispatches `WorkerIdle` and sleeps (`:261`); a real job goes to
`runJob` (`:251`/def `:494`) → `process` (`:534`). `process` raises `JobProcessing`
(`raiseBeforeJobEvent`, `:543`/`:805`), checks max attempts, and calls `$job->fire()`
(`:555`); `Job::fire()` (`Job.php:96`) parses the handler with `JobName::parse`
(`JobName.php:15`), resolves it from the container (`Job::resolve`, `Job.php:264`), and calls
`$this->instance->{$method}($this, $payload['data'])` (`:101`). On success `process` raises
`JobProcessed` (`:557`/`:819`) and, in its `finally`, `JobAttempted` (`:566`). If the job
throws, `handleJobException` (`:578`) runs the max-attempts / max-exceptions / no-retry
checks (`:659`/`:680`/`:706`), raises `JobExceptionOccurred` (`:609`/`:834`), and — if the job
is not deleted, released, or failed — computes `calculateBackoff` (`:747`), calls
`$job->release($backoff)` (`:623`), and dispatches `JobReleasedAfterException` (`:625`); then
re-throws (`:630`). Permanent failure runs `Job::failed` (`Job.php:247`, calls the handler's
`failed`), marks the job failed, and dispatches `JobFailed`. The loop's
`stopIfNecessary` (`:379`) stops on memory limit, queue-restart signal, job/time/idle
limits, or a lost connection (`stopWorkerIfLostConnection`, `:517`), exiting via `stop`
(`:961`).
