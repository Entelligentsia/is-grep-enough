# L3 reference answer key — inbound HTTP request → controller method

Pinned source: `experiment/repos/laravel` (SHA 2107d3d7…, PHP). All cites re-verified.

A single ordered chain across `src/Illuminate/Foundation/Http/Kernel.php`,
`src/Illuminate/Routing/Router.php`, `src/Illuminate/Routing/Route.php`, and
`src/Illuminate/Routing/ControllerDispatcher.php`. Order is gradeable.

## Required spine (gradeable core — must hit all, in order)

1. **Kernel entry** — `Kernel::handle($request)` `src/Illuminate/Foundation/Http/Kernel.php:137`
   calls `sendRequestThroughRouter($request)` (`:144`).
2. **Hand off into the middleware pipeline + router** — `Kernel::sendRequestThroughRouter`
   `Kernel.php:164` builds a `Pipeline` sending `$request` `through` the global middleware and
   `then($this->dispatchToRouter())` (`:174-178`); `dispatchToRouter()` (`:195`) returns a
   closure that calls `$this->router->dispatch($request)` (`:204`).
3. **Router dispatch entry** — `Router::dispatch(Request $request)`
   `src/Illuminate/Routing/Router.php:749` calls `dispatchToRoute($request)` (`:752`).
4. **Find the matching route** — `Router::dispatchToRoute` `Router.php:762` calls
   `runRoute($request, $this->findRoute($request))` (`:764`); `findRoute` (`:770`) dispatches a
   `Routing` event and calls `$this->routes->match($request)` (`:776`) to resolve the route.
5. **Run the route (route-matched event + stack)** — `Router::runRoute` `Router.php:793` sets
   the route resolver and dispatches `RouteMatched` (`:797-799`), then calls
   `runRouteWithinStack($route, $request)` (`:802`).
6. **Route middleware stack → `$route->run()`** — `Router::runRouteWithinStack` `Router.php:811`
   builds a `Pipeline` over the route's gathered middleware and `then`s a closure that calls
   `$route->run()` (`:821-824`).
7. **Route action dispatch** — `Route::run()` `src/Illuminate/Routing/Route.php:209` branches:
   a controller action → `runController()` (`:215`), else `runCallable()` (`:218`).
8. **Controller dispatcher** — `Route::runController()` `Route.php:273` calls
   `$this->controllerDispatcher()->dispatch($this, $this->getController(), $this->getControllerMethod())`
   (`:275-277`); `ControllerDispatcher::dispatch` `src/Illuminate/Routing/ControllerDispatcher.php:38`
   resolves method parameters (`:43`) and, if the controller has `callAction`, calls
   `$controller->callAction($method, $parameters)` (`:46`).
9. **Controller method invocation** — `Controller::callAction($method, $parameters)`
   `src/Illuminate/Routing/Controller.php:52` finally calls `$this->{$method}(...array_values($parameters))`
   (`:54`).

## Acceptable extras (credit, never required)

- The `Pipeline` "onion" semantics (global middleware in `sendRequestThroughRouter`, route
  middleware in `runRouteWithinStack`) and `gatherRouteMiddleware` / `resolveMiddleware`
  (`Router.php:829`/`:837`) — credit.
- `Route::runCallable()` (`Route.php:230`) using `CallableDispatcher::dispatch` for
  Closure/array actions — the non-controller branch; a plus.
- The `Routing` / `RouteMatched` / `RequestHandled` events (`Router.php:774`/`:799`,
  `Kernel.php:150`) fired along the way — credit.
- `findRoute` setting `Route::class` in the container (`Router.php:780`) and
  `$route->setContainer` (`:778`) — credit.
- `ControllerDispatcher::resolveParameters` / `resolveClassMethodDependencies`
  (`ControllerDispatcher.php:50`/`:53`) doing route-parameter + dependency injection — credit.

## Common misses / wrong turns

- Jumping straight to `Router::dispatch` or `Route::run` without the kernel front end
  (`Kernel::handle` → `sendRequestThroughRouter` → `dispatchToRouter`) — the prompt asks "from
  the moment the kernel takes it."
- Stopping at `Router::dispatch` / `runRoute` and never reaching the terminal controller
  method invocation (`Controller::callAction` → `$this->{$method}(...)`, `Controller.php:54`).
- Confusing the *global* middleware pipeline (`sendRequestThroughRouter`) with the *route*
  middleware pipeline (`runRouteWithinStack`); both are `Pipeline`s and the agent must
  distinguish them as two stops.
- Naming `dispatchToRouter` as the terminal step — it is a closure that calls
  `Router::dispatch`; the chain continues well past it.
- Treating it as a star of helpers rather than an ordered chain.

## Completeness scale

- **Full:** all 9 spine stops, in order, kernel → pipeline(global) → router dispatch →
  findRoute → runRoute → pipeline(route) → `Route::run` → `runController` →
  `ControllerDispatcher::dispatch` → `Controller::callAction` → method, with cites; the
  terminal `$this->{$method}(...)` named.
- **Partial:** ≥6 of 9 in roughly correct order, but missing either the kernel front end OR
  the terminal controller invocation, OR leaving the middleware-pipeline step vague.
- **Miss:** only the routing half, or an out-of-order/incorrect chain, or stopping at
  `Router::dispatch`.

## Model answer

`Kernel::handle($request)` (`Kernel.php:137`) calls `sendRequestThroughRouter($request)`
(`:144`), which builds a `Pipeline` sending the request through the global middleware and
`then($this->dispatchToRouter())` (`:174-178`); that closure (`dispatchToRouter`, `:195`)
calls `$this->router->dispatch($request)` (`:204`). `Router::dispatch` (`Router.php:749`)
delegates to `dispatchToRoute` (`:762`), which calls `runRoute($request, $this->findRoute($request))`
(`:764`); `findRoute` (`:770`) fires a `Routing` event and matches the request against the
route collection (`$this->routes->match($request)`, `:776`). `runRoute` (`:793`) sets the
route resolver, fires `RouteMatched`, and calls `runRouteWithinStack` (`:802`), which wraps
`$route->run()` in a second `Pipeline` over the route's gathered middleware (`:811-824`).
`Route::run()` (`Route.php:209`) branches to `runController()` (`:215`) for controller
actions; `runController` (`:273`) calls `ControllerDispatcher::dispatch($route, $controller,
$method)` (`:275-277`), which (`ControllerDispatcher.php:38`) resolves the method's
parameters and calls `$controller->callAction($method, $parameters)` (`:46`). Finally
`Controller::callAction` (`Controller.php:52`) invokes `$this->{$method}(...array_values($parameters))`
(`:54`) — the actual controller method. (Closure/array actions instead take
`Route::runCallable` → `CallableDispatcher::dispatch`, `Route.php:230`.)
