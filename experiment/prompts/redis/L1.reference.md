## Model answer

A single Redis value lives in `struct redisObject` (`object.h:100`), also typed `robj`
(115). Its first field `type:4` (101) records the logical data type (OBJ_STRING, OBJ_LIST,
‚Ä¶), while a separate `encoding:4` (102) records how that type is concretely represented
right now (e.g. OBJ_ENCODING_INT vs EMBSTR vs RAW for a string; LISTPACK vs QUICKLIST for a
list ‚Äî constants at 75-88) ‚Äî so type is the "what" and encoding is the "how." `refcount`
(103, width 23 at 95) manages lifetime via reference counting, with sentinel values for
permanently shared (`OBJ_SHARED_REFCOUNT` 96) and stack-allocated (`OBJ_STATIC_REFCOUNT` 97)
objects. `lru` (108, 24 bits, `LRU_BITS` 90) holds either an LRU clock stamp or packed LFU
access data, which the eviction machinery reads. Finally `ptr` (111) points at the actual
payload (SDS, dict, quicklist, ‚Ä¶) selected by `encoding`. When the object also serves as a
keyspace entry (`iskvobj` 104) it additionally carries `metabits` (107) and the key.
