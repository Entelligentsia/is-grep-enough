# L4 reference answer key — the template compile-and-render subsystem

Pinned source: `experiment/repos/django` (SHA 7903ee10…). All cites re-verified.
Paths are relative to the `django/` package dir; the subsystem lives in `template/base.py`.
A bounded cluster with two cooperating phases (compile, render) plus a node-type hierarchy.
Covering all four pieces matters more than strict line order within each.

## Required spine (gradeable core — must hit all four pieces)

1. **Compile is kicked off at construction** — `Template.__init__` `template/base.py:143` stores
   the source and sets `self.nodelist = self.compile_nodelist()` (`template/base.py:158`).
   `compile_nodelist` `template/base.py:179` builds a `Lexer` (or `DebugLexer`) and a `Parser`
   and runs them.
2. **Lex: source text → tokens** — `Lexer` `template/base.py:407`; `compile_nodelist` calls
   `lexer.tokenize()` (`template/base.py:191`, def `template/base.py:419`). `tokenize` splits on
   `tag_re` and turns each chunk into a `Token` via `create_token`
   (`template/base.py:428`, def `433`), classifying it as TEXT / VAR / BLOCK / COMMENT
   (`template/base.py:457-465`).
3. **Parse: tokens → node tree** — `Parser` `template/base.py:502`; `compile_nodelist` calls
   `parser.parse()` (`template/base.py:200`, def `template/base.py:530`). `parse` walks tokens
   and builds a `NodeList` (`template/base.py:541`): TEXT → `TextNode` (`template/base.py:548`),
   VAR → `VariableNode(filter_expression)` (`template/base.py:558`), BLOCK → the registered tag's
   compile function produces a node (`template/base.py:595-604`).
4. **Render: node tree + context → string** — `Template.render` `template/base.py:169` (binds the
   context, then `self._render` `template/base.py:166` → `self.nodelist.render(context)` 167).
   `NodeList.render` `template/base.py:1107` joins `node.render_annotated(context)` for every node
   into a `SafeString`. `Node.render_annotated` `template/base.py:1061` delegates to each node's
   `render` (`Node.render` `template/base.py:1055`). A variable placeholder is a `VariableNode`
   (`template/base.py:1156`) whose `render` (`template/base.py:1165`) resolves the
   `filter_expression` against the context and emits the value (TextNode just returns its literal
   text, `template/base.py:1127`).

## Acceptable extras (credit, never required)

- `render_value_in_context` (`template/base.py:1140`) and the auto-escaping it applies to a
  variable's value — credit if mentioned, but the escape *decision* is L2's territory; here it
  is the tail of `VariableNode.render`.
- `DebugLexer` (`template/base.py:468`) annotating tokens with positions when `engine.debug`.
- `Token`/`TokenType` (TEXT/VAR/BLOCK/COMMENT) classification detail.
- `Engine` / `Engine.get_default()` (`template/engine.py`) and `from_string` as the usual entry
  that constructs a `Template` — context, not required.
- `extend_nodelist` (`template/base.py`) and the `command_stack` used for block-tag error
  reporting; `get_exception_info` (`template/base.py:216`) for debug annotation.

## Common misses / wrong turns

- Collapsing compile and render into one step — the subsystem's structure is precisely the two
  phases: source→tokens→nodelist at construction, then nodelist→string at render time.
- Stopping at "it parses the template" without the `NodeList`/`Node`/`render_annotated`
  rendering side, or vice-versa (render without the lex/parse origin).
- Missing the token classification (TEXT/VAR/BLOCK) that determines which node type each token
  becomes.
- Not naming how a variable placeholder is rendered (`VariableNode.render` resolving a filter
  expression) — the prompt asks for it explicitly.
- Describing the `{% %}` tag library / `@register.tag` plumbing as the whole subsystem — it is
  one branch of `parse` (BLOCK tokens), not the spine.

## Completeness scale

- **Full:** all four pieces — compile kicked off in `__init__`, `Lexer.tokenize` producing
  classified tokens, `Parser.parse` building a `NodeList` of typed nodes, and
  `Template.render` → `NodeList.render` → per-node `render_annotated`/`render` (incl. how a
  `VariableNode` emits its value) — connected with cites, distinguishing the compile vs render
  phases.
- **Partial:** 3 of 4 (commonly missing the token classification, or the per-node render detail).
- **Miss:** only "parses and renders" with no account of tokens, the node tree, or the render
  traversal.

## Model answer

A `Template` is compiled the moment it is built: `Template.__init__` (`template/base.py:143`)
saves the source and sets `self.nodelist = self.compile_nodelist()` (158). `compile_nodelist`
(179) constructs a `Lexer` (407) and runs `lexer.tokenize()` (191/def 419), which splits the
source on `tag_re` and turns each piece into a `Token` via `create_token` (428/433), tagging it
TEXT, VAR, BLOCK or COMMENT (457-465). Those tokens go to a `Parser` (502); `parser.parse()`
(200/def 530) iterates them into a `NodeList` (541): TEXT becomes a `TextNode` (548), a `{{ }}`
variable becomes a `VariableNode` wrapping a compiled filter expression (558), and a `{% %}`
block is handed to its registered tag's compile function to produce a node (595-604). Rendering
is the second phase: `Template.render` (169) binds the context and calls `_render` (166) →
`self.nodelist.render(context)` (167); `NodeList.render` (1107) concatenates
`node.render_annotated(context)` (def 1061, which wraps each node's `render`, 1055) for every
node into a `SafeString`. A `TextNode` just returns its literal text (1127); a `VariableNode`
(1156) `render` (1165) resolves its filter expression against the context (and runs the value
through `render_value_in_context`, 1140, for auto-escaping) to produce its output fragment. So
the subsystem is: source → tokens (lex) → node tree (parse) at construction, then node tree +
context → string (render).
