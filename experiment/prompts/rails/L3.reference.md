# L3 reference answer key ‚Äî query materialization (forced load ‚Üí SQL ‚Üí rows ‚Üí model objects)

Pinned source: `experiment/repos/rails` (SHA abbcb8d7‚Ä¶). All cites re-verified.

A single ordered chain across `activerecord/lib/active_record/relation/query_methods.rb`,
`relation.rb`, `querying.rb`, `connection_adapters/abstract/database_statements.rb`, and
`persistence.rb`. Order is gradeable. (The simple, non-eager-load path is the spine; the eager-load
branch is an acceptable extra.)

## Required spine (gradeable core ‚Äî must hit all, in order)

1. **Build the where-clause (predicate)** ‚Äî `Relation#where` `query_methods.rb:1097` calls
   `spawn.where!(*args)` (`:1105`); `where!` (`:1107`) does `self.where_clause +=
   build_where_clause(opts, rest)` (`:1108`), which routes values through
   `PredicateBuilder#build` (`predicate_builder.rb:57`) to produce Arel bind nodes. This only mutates
   the relation ‚Äî no SQL runs yet.
2. **Force materialization** ‚Äî `Relation#load` `relation.rb:1205` calls `exec_queries`
   (`relation.rb:1429`) when not loaded (the trigger that turns the built relation into a query).
3. **Dispatch to the main query** ‚Äî `exec_queries` calls `exec_main_query` (`relation.rb:1442`);
   on the simple path `exec_main_query` does `model.with_connection { |c| model._query_by_sql(c,
   arel, async: async) }` (`:1474-1476`).
4. **Hand SQL to the connection** ‚Äî `_query_by_sql` `querying.rb:79` calls
   `connection.select_all(sanitize_sql(sql), "#{name} Load", ‚Ä¶)` (`:81`). `select_all`
   (`database_statements.rb:71`) wraps the arel in a `QueryIntent` (`:72-80`) and calls
   `intent.execute!` (`:82`) to run the SQL, then `intent.cast_result` (`:88`) to produce the
   row set. (The adapter's `to_sql`/`to_sql_and_binds`, `database_statements.rb:17`, turns the Arel
   tree into the final SQL string inside the intent.)
5. **Instantiate rows into records** ‚Äî back in `exec_queries`, `instantiate_records(rows)`
   (`relation.rb:1485`) is called; on the simple path it calls `model._load_from_sql(rows)`
   (`relation.rb:1495`). `_load_from_sql` (`querying.rb:83`) iterates `result_set.indexed_rows` and
   maps each row through `instantiate(record, column_types)` (`querying.rb:108`).
6. **Build the model instance** ‚Äî `ActiveRecord.instantiate` `persistence.rb:100` picks the STI
   subclass via `discriminate_class_for_record` (`:101`) and calls `instantiate_instance_of`
   (`:102`) to allocate the record and assign the column attributes ‚Äî the terminal step that yields
   the in-memory ActiveRecord object.

## Acceptable extras (credit, never required)

- The eager-loading branch inside `exec_main_query` (`relation.rb:1456-1470`): when
  `eager_loading?` it uses `apply_join_dependency` and `c.select_all(relation.arel, "‚Ä¶ Eager Load",
  ‚Ä¶)` instead of `_query_by_sql`. Mentioning it is a plus; the simple path above is the required
  spine.
- `QueryIntent`/`FutureResult` async path (`select_all` `async:` branch returns
  `intent.future_result` `:86`) ‚Äî `load_async`/`scheduled?` collects it in `exec_queries`
  (`relation.rb:1444-1446`).
- `instantiate_instance_of` / the `@attributes` `AttributeSet` construction that
  `instantiate_instance_of` drives (the bridge from a raw row hash to typed `ActiveModel::Attribute`
  objects ‚Äî ties this chain to the L1 container).
- The `ConnectionAdapters::AbstractAdapter#to_sql` (`database_statements.rb:12`) /
   `arel_from_relation` (`:802`) step that compiles the Arel `SelectManager` to a SQL string.
- Bind parameter handling / `to_sql_and_binds` (`:17`) and prepared-statement caching.

## Common misses / wrong turns

- Stopping at `where`/`load` and never reaching the connection (`select_all`) or the instantiation
  ‚Äî skips the back half of the chain; the prompt asks end-to-end.
- Confusing `exec_main_query` with the eager-load branch, or citing the eager path as the only path.
- Naming `exec_query`/`execute` (`database_statements.rb:162`/`:175`) instead of `select_all`
  (`:71`) ‚Äî those are lower-level primitives; the relation path goes through `select_all`/`QueryIntent`.
- Ending the chain at "rows come back" without the `instantiate`/`instantiate_instance_of` step that
  produces actual model objects (the prompt's stated endpoint).
- Treating it as an unordered set of functions rather than a directed sequence.

## Completeness scale

- **Full:** all 6 spine stops in order ‚Äî where-build ‚Üí load/exec_queries ‚Üí exec_main_query ‚Üí
  _query_by_sql/select_all (QueryIntent.execute!) ‚Üí instantiate_records/_load_from_sql ‚Üí
  instantiate ‚Äî with cites; the terminal `instantiate`/`instantiate_instance_of` named.
- **Partial:** ‚â•4 of 6 in roughly correct order, but missing either the connection/`select_all` step
  OR the final instantiation step, or with the SQL-handoff left vague.
- **Miss:** only the query-building half (`where`/`load`), or an out-of-order/incorrect chain.

## Model answer

`where(active: true)` only builds the relation: `Relation#where` (`query_methods.rb:1097`) calls
`spawn.where!` (`:1105`), which appends to `where_clause` via `build_where_clause` ‚Üí
`PredicateBuilder#build` (`predicate_builder.rb:57`), producing Arel bind nodes ‚Äî no SQL runs. The
query fires when something forces `load` (`relation.rb:1205`), which calls `exec_queries`
(`:1429`). `exec_queries` calls `exec_main_query` (`:1442`); on the simple path that does
`model._query_by_sql(c, arel)` (`:1474`). `_query_by_sql` (`querying.rb:79`) hands the arel to
`connection.select_all` (`:81`); `select_all` (`database_statements.rb:71`) builds a `QueryIntent`
and calls `intent.execute!` (`:82`) to run the SQL (compiled to a string by the adapter's
`to_sql`/`to_sql_and_binds`, `:17`) and then `intent.cast_result` (`:88`) to return the rows. Back
in `exec_queries`, `instantiate_records(rows)` (`relation.rb:1485`) calls `model._load_from_sql`
(`:1495`), which (`querying.rb:83`) maps each row through `instantiate(record, column_types)`
(`:108`). `ActiveRecord.instantiate` (`persistence.rb:100`) resolves the STI subclass with
`discriminate_class_for_record` (`:101`) and calls `instantiate_instance_of` (`:102`) to build the
typed record ‚Äî the end of the chain, raw rows now live as ActiveRecord objects. (When
`eager_loading?` is set, `exec_main_query` instead joins + `select_all`s the eager relation; that's
the parallel branch, not the main spine.)
