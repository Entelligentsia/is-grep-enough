# L3 reference answer key — the build→render→write chain (CLI invocation → page written to disk)

Pinned source: `experiment/repos/hugo` (SHA d15baf53…, Go). All cites re-verified.

A single ordered chain across `commands/` and `hugolib/`. Order is gradeable. This is the **full-build**
branch (no file events); the incremental-rebuild branch is a separate concern (see the L5 key).

## Required spine (gradeable core — must hit all, in order)

1. **Command entry** — `hugoBuildCommand.Run` `commands/commands.go:71` → `c.rootCmd.Run`
   `commands/commandeer.go:393`.
2. **Builder dispatch** — `rootCommand.Run` creates `b := newHugoBuilder(r, nil)` (`commandeer.go:394`) and
   calls `b.build()` `commands/hugobuilder.go:410`.
3. **fullBuild → buildSites** — `hugoBuilder.build` → `c.fullBuild(false)` `hugobuilder.go:422` →
   `c.buildSites(noBuildLock)` `hugobuilder.go:440`.
4. **Into the engine** — `buildSites` calls `h.Build(hugolib.BuildCfg{NoBuildLock: noBuildLock})`
   `hugobuilder.go:450`.
5. **`HugoSites.Build` — the phase orchestrator** — `hugolib/hugo_sites_build.go:64`. It runs, in order:
   `h.process(ctx, infol, conf, init, events...)` `:171` → `h.assemble(ctx, infol, conf)` `:177` →
   `h.render(infol, conf)` `:197` → `h.writeBuildStats()` `:203` → `h.renderDeferred(infol)` `:212` →
   `h.postProcess(infol)` `:221`. (Defs: `process` `:283`, `assemble` `:302`, `render` `:399`,
   `renderDeferred` `:488`, `postProcess` `:650`.) With no events, `process` takes the `h.processFull`
   branch (`hugo_sites_build.go:297`).
6. **render phase fans out per site / output format** — `h.render` `hugo_sites_build.go:399` loops sites and
   `renderFormats`, calls `s.preparePagesForRender(...)` (`:453`) then, on the non-partial path,
   `s.render(siteRenderContext)` (call `:469`; def `hugolib/site.go:1787`).
7. **`Site.render` → renderPages** — `Site.render` `site.go:1787` → `s.renderPages(ctx)`
   `hugolib/site_render.go:71` (def `:71`).
8. **renderPages spawns workers over the page tree** — `renderPages` `site_render.go:71` starts
   `numWorkers` goroutines running `pageRenderer` (`:82-87`) and a `doctree.NodeShiftTreeWalker` over
   `s.pageMap.treePages` whose `Handle` feeds each `*pageState` onto a `pages` channel (`:89-110`).
9. **pageRenderer, per page** — `pageRenderer` `site_render.go:123`: `p.renderResources()` (`:148`),
   optional alias rendering `s.renderAliasesForPage(p)` (`:161`), then `templ, found, err :=
   p.resolveTemplate()` `site_render.go:176`.
10. **Template resolution (treated as a black box here)** — `pageState.resolveTemplate`
    `hugolib/page.go:630` builds a `tplimpl.TemplateQuery{Category: CategoryLayout, ...}` (`page.go:638-643`)
    and calls `ps.s.TemplateStore.LookupPagesLayout(q)` `page.go:645`, returning a `*tplimpl.TemplInfo`.
11. **Write** — `s.renderAndWritePage(&s.PathSpec.ProcessingStats.Pages, targetPath, p, d, templ)`
    `site_render.go:212` (def `hugolib/site.go:1647`).
12. **Execute + publish** — `renderAndWritePage` `site.go:1647` calls
    `s.renderForTemplate(ctx, p.Kind(), of.Name, d, renderBuffer, templ)` `site.go:1658` (def `site.go:1743`),
    which calls `s.GetTemplateStore().ExecuteWithContext(ctx, templ, w, d)` `site.go:1753`; then builds a
    `publisher.Descriptor` `site.go:1669-1673` and calls `s.publisher.Publish(pd)` `site.go:1695`.

## Common misses / wrong turns

- Stopping at `HugoSites.Build` without naming the ordered phase sequence
  (process→assemble→render→renderDeferred→postProcess).
- Skipping the command/builder front end (`commands/`) and starting at `hugolib` — the prompt says "from
  invoking a Hugo build on the command line."
- Naming `renderPages` but not the worker / tree-walk feeding `pageRenderer`.
- Stopping at `resolveTemplate` and not following through to `ExecuteWithContext` + `publisher.Publish` — the
  prompt explicitly asks "executed and its result written out."
- Confusing `renderForTemplate` (the execute wrapper, `site.go:1743`) with the page-render entry — the
  execute wrapper handles the nil-template "missing layout" log path (`:1745-1748`).

## Completeness scale

- **Full:** the ordered chain from command entry (`hugoBuildCommand.Run`) through `Build`'s phase sequence,
  `renderPages` + workers, `pageRenderer`, `resolveTemplate`, `renderAndWritePage` → `renderForTemplate` →
  `ExecuteWithContext` → `publisher.Publish`, with cites.
- **Partial:** most stops in order but missing either the command front end OR the execute+write terminal,
  or with the phase sequence out of order.
- **Miss:** only `hugolib` internals, or `renderPages` without the front end and the write terminal.

## Model answer

Running `hugo build` hits `hugoBuildCommand.Run` (`commands/commands.go:71`) → `rootCommand.Run`
(`commandeer.go:393`), which builds a `hugoBuilder` (`:394`) and calls `b.build()` (`hugobuilder.go:410`) →
`fullBuild(false)` (`:422`) → `buildSites` (`:440`) → `h.Build(hugolib.BuildCfg{...})` (`:450`). `HugoSites.Build`
(`hugo_sites_build.go:64`) runs its lifecycle phases in order: `process` (`:171`) → `assemble` (`:177`) →
`render` (`:197`) → `renderDeferred` (`:212`) → `postProcess` (`:221`). The `render` phase (`:399`) loops
sites/output-formats and calls `s.render(ctx)` (`site.go:1787`) → `s.renderPages` (`site_render.go:71`),
which starts `pageRenderer` workers (`:123`) fed by a tree-walk over `s.pageMap.treePages`. Each
`pageRenderer` renders resources, then resolves the page's template via `p.resolveTemplate()` (`page.go:630`
→ `TemplateStore.LookupPagesLayout`, `:645`) and calls `s.renderAndWritePage(...)` (`site_render.go:212` /
def `site.go:1647`). `renderAndWritePage` runs `renderForTemplate` (`site.go:1658`/`1743`) →
`ExecuteWithContext` (`:1753`) to fill a buffer, wraps it in a `publisher.Descriptor` (`:1669-1673`), and
finally `s.publisher.Publish(pd)` (`:1695`) writes the page to disk.
