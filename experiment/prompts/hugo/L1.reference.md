## Model answer

A single Hugo media type lives in `media.Type` (`media/mediaType.go:36`). Its `Type` field (`mediaType.go:38`)
holds the full MIME identity string, e.g. `"application/rss+xml"`. That identity is decomposed into
`MainType` (`:41`, e.g. `application`) and `SubType` (`:43`, e.g. `rss`), with an unexported `mimeSuffix`
(`:52`) holding the optional structured-suffix tail (e.g. `xml` in `application/rss+xml`) — so `SubType` is the
MIME subtype proper and `mimeSuffix` is the optional `+tail`, not the same thing. The file-suffix
representation is `Delimiter` (`:45`, default `"."`), `FirstSuffix SuffixInfo` (`:48`) — whose `Suffix`
(`SuffixInfo` `:61` / `:63`) is the bare extension and `FullSuffix` (`:66`) includes the delimiter — and
`SuffixesCSV` (`:57`), the alternate extensions joined as a comma string so the struct stays comparable and
map-key-usable. Built-in media types are constructed in `media/builtin.go` (e.g. `HTMLType` `:10` /
`RSSType` `:18`, populated in `var Builtin` `:62`).
